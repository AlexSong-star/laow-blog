## 3张图读懂今天的 OpenClaw

### 图1：OpenClaw 登顶 GitHub，20年来首个非语言/浏览器项目

3月2日，OpenClaw 正式超越 React，成为 GitHub 全站 star 数量第一的软件项目。

这不是偶然——React 当年靠的是"前端革命"；OpenClaw 靠的是"AI 原生"的技术范式。GitHub CEO 在声明中说：*"OpenClaw 代表了 2010 年代移动化之后最大的开发者行为转变。"*

关键数字：
- OpenClaw 项目数：48,000+
- 注册开发者：210万
- 活跃 agent：周活跃 3800万

---

### 图2：安全生态大爆发——5款工具同时涌现

过去30天，至少5款 OpenClaw 安全类工具/技能上线：

| 工具 | 定位 | 核心功能 |
|------|------|----------|
| **ClawCare** | 运行时安全扫描 | agent skill 恶意代码扫描 |
| **Agent Passport** | 身份认证 | OAuth 风格的 agent 身份验证 |
| **ClawShield** | 通信防火墙 | agent 间通信的流量过滤 |
| **Telos** | 运行时安全 | eBPF/LSM 内核级防护 |
| **RunVeto** | 紧急开关 | 一键终止失控 agent |

这像极了 2014-2015 年 Docker 生态爆发时的安全工具潮——先跑量，再补安全。

---

### 图3：Google 封杀 OpenClaw 用户——这其实是最好的广告

2月22日，Google 更新服务条款，限制 AI Pro/Ultra 订阅者使用第三方 AI agent 框架访问其服务。

802 个 HN points，大量讨论——这不是 OpenClaw 的危机，这是 Google 在用政策手段对抗技术趋势。

HN 热评：*"你封杀的不是一个工具，你封杀的是用户对你平台的依赖。"*

---

## 快速评论

OpenClaw 用 200 天完成的事情，React 用 5 年：

1. **框架→生态**：从工具到平台，第三方 skill 市场已收录 12,000 个技能
2. **安全先行**：不等出事了再补，生态在生长过程中同步长出安全层
3. **企业采用**：EZClaw（1分钟部署）的出现意味着，非技术背景的企业主也能用 AI agent 了

OpenClaw 的故事，本质上是一个关于**信任**的故事——开发者信任开源社区，企业主信任低门槛工具，用户信任 agent 能替他们做事。

200天，刚开始。
