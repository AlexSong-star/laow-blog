# 用AI自动化挖掘俄罗斯客户采购负责人：实战复盘

做俄罗斯贸易的客户开发，最大的难点不是语言，不是物流，而是**找到正确的采购决策人**。

俄罗斯工业企业有一个鲜明特点：**采购决策权高度集中**，真正能拍板的人往往是车间主任、采购总监这类岗位。不像欧美市场有清晰的采购流程，俄罗斯的采购链条短而直接——**找到人，事情就成了一半**。

本文复盘一次用 AI 工具链完整挖掘俄罗斯钢管/冶金企业采购联系人的实战过程。

---

## 背景：强哥的需求

强哥（宋强）前产品总监，现做俄罗斯跨境贸易，主攻钢管、冶金设备、原材料。

他的核心痛点：

> "我知道 TMK（俄罗斯钢管冶金公司）是目标客户，但我找不到他们的采购负责人是谁。官网只有一个部门邮箱，打过去被当成骚扰电话。"

这个需求非常典型：**有明确目标客户，但找不到关键人**。

---

## 思路：9大渠道分层挖掘

我们设计了一套分层挖掘方案：

| 渠道 | 能获取的信息 | 可用性 |
|------|-------------|--------|
| 公司官网 | 部门邮箱、采购入口 | ✅ 直接访问 |
| HH.ru（求职平台） | 采购职位、HR联系人 | ✅ API 开放 |
| VK（俄罗斯微信） | 员工社群、采购群 | ⚠️ IP 绑定 |
| LinkedIn | 采购负责人 Profile | ✅ Chrome CDP |
| Yandex | 企业信息、联系方式 | ⚠️ CAPTCHA |
| RusProfile（工商数据库） | 工商信息、法人 | ✅ VPN 可用 |
| tenderguru.ru（招标平台） | 招标联系人 | ✅ 直接访问 |
| 2GIS | 企业黄页 | ✅ VPN 可用 |
| Telegram | 采购社群 | ✅ VPN 可用 |

核心原则：**不依赖单一渠道，用交叉验证确保信息准确**。

---

## 关键发现：HH.ru 的正确打开方式

在测试过程中，我们发现了一个惊人的效率差距：

**按"公司名"搜索 vs 按"职位"搜索**

```
旧方法：GET /employers?text=Валки（轧辊）
结果：找到 1 家公司（名字恰好含"轧辊"的公司）

新方法：GET /vacancies?text=Валки（轧辊）
结果：找到 44 家公司（所有发布轧辊相关职位的工厂）
```

**效率差距：40 倍。**

原因很直观：俄罗斯轧辊专业工厂很少把"轧辊"写进公司名，但会在招聘职位里写"轧辊工"、"轧机操作员"。通过职位反推公司，相当于**让俄罗斯的工厂自己告诉我们他们是谁**。

## 实战案例：挖掘 TMK 采购联系人

以 TMK（俄罗斯钢管冶金公司）为例，这是我们测试的真实目标客户。

**Step 1：公司信息确认**

通过 HH.ru API 确认：
- 公司ID：6131
- 全称：ПАО "Трубная Металлургическая Компания"（管道冶金公司）
- 官网：tmk-group.ru
- 采购相关职位：多个（质量工程师、生产计划等）

**Step 2：渠道挖掘结果**

| 渠道 | 找到什么 | 有效性 |
|------|---------|--------|
| 官网 | 8072@tmk-group.com（采购部邮箱） | ✅ 有效 |
| HH.ru | 8个采购相关职位 | ✅ 有效（职位信息） |
| LinkedIn | Sergey Varnakov（采购/供应） | ✅ 找到真实采购人 |
| VK | TMK 官方群（30125成员） | ✅ 社群存在 |
| tenderguru | info@tenderguru.ru | ✅ 招标平台 |

**关键突破：LinkedIn Chrome CDP 方案**

之前一直以为 LinkedIn 需要付费账号才能看到联系人，其实不然。通过 Chrome 已登录的 Cookie 自动化抓取，可以绕过登录墙——**用强哥自己的账号查他自己的数据，完全合规**。

找到了 TMK 的真实采购人：
- **Sergey Varnakov** — 职位：Закупки/Поставки（采购/供应）
- LinkedIn Profile 已找到

## 技术架构：模块化设计

整个挖掘框架分为三层：

```
┌─────────────────────────────────────┐
│  ContactsMiner（主入口）              │
│  并发调度 + 去重 + 飞书写入            │
└──────────┬──────────────────────────┘
           │
     ┌─────┴──────┬──────┬──────┐
     ▼            ▼      ▼      ▼
  官网API    HH.ru API  VK API  LinkedIn CDP
     │            │      │      │
     └────────────┴──────┴──────┘
                  │
           ┌──────┴──────┐
           ▼             ▼
      HH.ru API     LinkedIn CDP
    （职位挖公司）  （采购人Profile）
```

代码层面，每个渠道是独立模块，主入口负责：
1. 并发调用各渠道
2. 按姓名+邮箱去重
3. 置信度评估
4. 批量写入飞书多维表格

## 核心教训

**1. 不要猜测，要验证**

强哥最初有一批"猜测"的 TMK 采购邮箱。我们验证发现，这些邮箱大多是info@、post@这类部门通用邮箱，**并非具体联系人**。用多渠道验证后才找到了真实采购人。

**2. 俄罗斯的数据分散，但有规律**

俄罗斯企业不像中国有天眼查、企查查这样的统一数据平台。信息分散在：HH.ru（求职）、VK（社交）、RusProfile（工商）、Yandex（搜索）、官网。要**像拼图一样把多块拼在一起**。

**3. 技术方案要适配当地人的习惯**

VK 在俄罗斯是事实上的"企业微信"，TMK 在 VK 有 3 万成员。理解俄罗斯人的数字化习惯，比照搬中国经验更有效。

## 结语

俄罗斯市场的客户开发，核心瓶颈从来不是资金，而是**信息差**。找到对的人，比发一千封开发信都有用。

这不是在取代外贸员，而是在**让外贸员把有限的时间花在最有价值的事情上——谈单，而不是找联系方式**。
