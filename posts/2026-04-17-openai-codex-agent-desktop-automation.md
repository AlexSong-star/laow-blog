# AI开始操控你的电脑：OpenAI Codex桌面版重塑人机协作

上周，OpenAI发布了一个公告，很容易被淹没在AI新闻的信息洪流里：**Codex桌面版现在可以在后台操控你电脑上的所有应用程序**。

如果你没有意识到这意味着什么，让我用一句话说明白：**你电脑里每一个安装的应用，现在都可以被AI Agent直接操作——而你正在做别的事情**。

这不只是"更聪明的Siri"或"会写代码的Copilot"。这是一个**全新的计算范式**，而且它已经存在了。

## Codex能做什么

根据OpenAI在4月16日TDX大会上的演示，新版Codex可以：

**第一，后台操控所有应用。** 你在写文档，Codex在后台帮你：
- 检查Slack里有没有需要你处理的紧急消息
- 从Gmail里提取重要邮件摘要
- 查看Google Calendar告诉你今天的会议安排
- 在Notion里更新项目状态

所有这些，**你不需要切换窗口，不需要复制粘贴，Codex会在你需要的时候主动告诉你结果**。

**第二，90多个插件生态。** CircleCI、GitLab、Microsoft Suite、GitHub PR……Codex现在是一个**连接所有开发工具的中枢神经系统**，而不是一个孤立的聊天窗口。

**第三，内置浏览器 + 图像生成。** 你可以让Codex生成一个网站原型，它会直接生成图像素材、编写代码、在内置浏览器里预览——全部在一个环境里完成。

## 为什么这重要

过去60年，我们和电脑的交互模式几乎没有变过：**人类发出指令，电脑执行**。

鼠标、键盘、触摸屏——本质上都是人类在"驱动"电脑。即使是语音助手（Siri、Alexa），也不过是把手指换成了声音，本质上还是人类在发起动作。

Codex的新模式完全不同：**AI可以成为驱动者**。

它不是在等你告诉它做什么。它在后台监视、推理、行动。它会在你的待办事项出现冲突时主动协调，会在你忘记跟进时提醒你，会在你需要决策时把信息汇总好放在你面前。

这不是自动化脚本——脚本是死的，Codex是**活的有代理能力的**。

## "多Agent工作流"终于来了

演示中最令人震撼的场景：**你可以一边用VS Code写代码，一边让Codex同时帮你做这些事情**：

- 分类处理Slack里的10条客户消息
- 审查你昨天的代码PR
- 更新JIRA里相关工单的状态

这三个任务**并行进行**，而你完全没有被打断。

这就是传说中的"多Agent工作流"——多个AI Agent同时在不同任务上工作，而人类扮演的是**管理者**而不是执行者。

你不再是一个执行者。你是一个**指挥官**。

## Claude Code vs Codex：两条不同的路

有趣的是，Anthropic的Claude Code和OpenAI的Codex正在走向**完全不同的方向**。

Claude Code专注在**编程任务**上——代码补全、调试、重构，它是程序员的专属工具，强调的是"帮程序员写更好的代码"。

Codex的野心明显更大——它是把电脑本身变成AI可以操控的基础设施，不限于编程场景，而是**任何你用电脑做的事**。

一个在深度上精进，一个在广度上扩张。两条路都很合理，也都很可怕——可怕是因为，无论哪条路，都在重新定义"人操作电脑"这件事。

## 风险也在放大

当我写这篇文章的时候，我意识到一件不舒服的事：**如果AI可以操控你电脑上的所有应用，它也同时拥有相应的权限和风险**。

Slab Leak告诉我们：在AI时代，"影子IT"已经进化成了"影子AI"——员工在后台运行AI Agent处理工作，但IT部门完全不知道。

Codex的权限有多大？它能读取哪些数据？它会不会把你的隐私信息发送给第三方？

OpenAI说这些都是"企业级安全"，但坦率地说：**我们还不知道答案**。

这不是泼冷水。这是提醒：**在拥抱AI操控电脑的浪潮之前，我们需要先回答"信任谁"这个问题**。

## 你什么时候能用上它

好消息：Codex桌面版**现在就能用**。macOS和Windows都可以下载，基础功能已经对所有用户免费/付费用户开放。

后台计算机使用功能目前**仅限macOS**——这显然是第一步，之后会扩展到Windows。
